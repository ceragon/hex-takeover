# HQ-rush 为核心胜利路径，倒计时降级为防僵局兜底

实测 v2 在「摧毁 HQ 直胜」与「时间耗尽比格数」两个并列胜利条件下，HQ-rush 约 30 秒即可终结对局，180 秒倒计时几乎从不触发。经权衡，确定 **核心玩法 = 攻击敌方 HQ / 守住己方 HQ**：倒计时不再是并列的主胜负手，而是**防僵局兜底**——仅当双方都无力破家时，才按格数判胜。围绕「保护主堡」组织对局张力。

## Considered Options

- **(A) 三分钟领地拉锯（拒绝）**：以 180 秒倒计时为主线，比谁占格多，HQ-rush 只是高风险翻盘手段。拒绝理由：与「保护主堡」的核心张力背道而驰，且需要让 HQ 极难突破，反而削弱攻防博弈。
- **(B) HQ-rush 为核心（采纳）**：速攻破家即核心玩法，攻防双方都围绕主堡安危做决策。倒计时退为兜底。

## Consequences

- **AI 必须具备防守反击**：当前 `scoreAction` 给兵营 +4、金矿 +25，AI 几乎不建军、不对入侵反应，导致 30 秒一边倒。在 B 路线下这是必须修的核心缺陷，而非可选打磨。
- **倒计时时长留作平衡旋钮**：180s 是否缩短，待 AI 防守修好、对局自然拉长后再定；不在本 ADR 锁死。
- **HUD/文案应弱化倒计时、突出双方 HQ 安危**：倒计时作为兜底信息呈现，主视觉应让玩家时刻感知己方/敌方 HQ 的威胁程度（呼应 HQ 血条必须真实可读，见 HUD 读 `tile.building.hp` 的修复）。
