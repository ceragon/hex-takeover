# v2 架构转向：完全取代 v1 并重做核心循环

v1 的"迷雾翻牌 + 步骑弓三角克制"在机制深度和经济设计上已走到瓶颈，无法平滑演进到 v2 的"开地块 + 品阶压制"体系。经对照 `docs/research/gameplay-v2.md` 调研结果，决定 **v2 完全取代 v1**，不保留 v1 的 reveal 机制和克制链作为并列模式或兼容层。

v1 代码作为参考保留在 git 历史中（commit `2653aa2` 之前）。

## Considered Options

- **完全取代（采纳）**：删除 v1 机制，按 v2 重做。风险集中在 25 天 deadline 内，但单套规则便于调试、测试、平衡。
- **共存（拒绝）**：保留 v1 作为"经典模式"，v2 作为新模式。GameState 需支持两套规则分支，复杂度翻倍，单人项目 25 天内无法收敛。
- **渐进过渡（拒绝）**：在 v1 骨架上先让 v2 跑起来再打磨。实际上 v1 的 reveal 机制和 v2 的开地块机制在世界观层面冲突，没有可行的过渡路径。

## 重做范围

保留的骨架：Canvas 渲染、HexGrid 坐标数学、10Hz 游戏循环、GameState 单例模式、MuleRun 单文件交付。

重做的内脏：TileReveal → TileSystem（格子状态机）、BarracksSystem（兵营槽按 CD 吐兵）、Combat（品质压制取代克制链）、moveUnits（品阶差异化寻敌）、checkVictory（HQ 血量 + 3 分钟倒计时）、AIBehavior（评分-选择算法）、ResourceSystem（HQ/金矿双产金 + 开地块/升级双花费）。地图从 5×7 扩到 9×12 + 湖水行。

## 核心机制决议

- **玩家动作**：只有"开地块"一种核心动作（点击可开格子 → 扣费 → 走二次概率表出建筑）。
- **格子生命周期**：未激活 → 可开（显示图标 + 价格）→ 已开（显示具体建筑，不可逆）。
- **混合池权重**：兵营槽(?) 55% / 未确定兵种(头盔) 25% / 箭塔槽(盾牌) 10% / 金矿槽(金属) 10%。
- **价格档位差异化**：兵营槽偏 25 金（便宜赌局），头盔/箭塔槽偏 50/100 金（稳定），金矿槽偏 50 金。
- **25 金档兵营槽特例**：50% 概率"空"（扣费但不出建筑），这是设计上的高风险低成本选项。

## AI 策略

- **三档难度**（简单 / 普通 / 困难），MVP 实现普通 + 困难。
- **完全对等经济**：AI 与玩家付同样的钱、按同样的概率表出建筑，不作弊。难度差异靠"决策质量"调节，而非经济作弊。
- **评分-选择算法**：每 1.5 秒决策循环，枚举合法动作 → 启发式打分 → 按难度选择（困难选最优，普通从 top-3 随机选）。
- **全图可见**：AI 能看到玩家的所有建筑、单位、金币数，不实现 AI 视野限制。

拒绝的 AI 方案：minimax 博弈树（v2 动作空间小，启发式足够）、行为树（评分-选择更易调参）、AI 经济作弊（破坏玩家信任感）、AI 视野限制（实现复杂度高、收益小）。

## 开发方式

**垂直切片迭代**，5 个切片，每个切片保持"游戏能跑起来"：

1. 地图地基（9×12+湖水、TileSystem、开局 6 格、资源系统）
2. 产兵与推线（BarracksSystem、moveUnits、领地推线）
3. 战斗系统（Combat、箭塔、建筑 HP、废墟）
4. 胜负闭环（checkVictory、升级系统、UI）
5. AI 与打磨（AIBehavior、粒子特效）

拒绝的开发方式：一次性大重构（单文件 3000+ 行难调试）、水平分层重构（模块定义完接入 game loop 时容易接口不对）。

## Consequences

- **v1 测试全部失效**：9 个 embedded `console.assert` 测试随 v1 机制一起删除，v2 测试在重写过程中逐步补齐。
- **数值需要重新平衡**：v2 的 6 兵种、4 档开地块、3 档升级引入了远比 v1 复杂的经济网络，"待调优"数值占多数。切片 1 完成后即可手动试玩调优。
- **单文件体积膨胀**：预计从 v1 的 1369 行扩到 v2 的 2500-3000 行。仍可维护，但已接近单文件架构的合理上限。
- **Deadline 风险集中**：25 天完成 5 个切片 = 每切片 5 天，无缓冲。若某切片卡住需果断砍 scope 而非加 deadline。
