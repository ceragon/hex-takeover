# 转向竖屏移动端：540×960 固定逻辑分辨率 + 全画布 HUD

基于一张参考 UI 图，决定把交付目标从"桌面浏览器横版 560×540"改为**竖屏移动端**。这**推翻了 `AGENTS.md → Out of Scope` 中"mobile adaptation（移动端适配）"不做的决定**。该条目即日起作废，移动端竖屏成为主交付形态。

## Considered Options

### 屏幕方向
- **视觉换皮、保持横版（拒绝）**：把参考图只当画风参考，画布仍横版。最低风险，但与"我希望游戏 UI 是这样"的竖屏意图不符。
- **竖屏移动端（采纳）**：真正转向竖屏作为主目标。需重定画布尺寸、缩放、输入，但符合产品意图。
- **竖屏比例但仍桌面（拒绝）**：高瘦画布居中显示，不声称支持移动端。是半吊子，既不还原手机观感也不真支持手机。

### 缩放策略
- **固定逻辑分辨率 540×960 + letterbox（采纳）**：内部按 540×960（9:16）渲染，CSS `object-fit: contain` 等比放大，留黑边。游戏逻辑只面对单一坐标系，美术稿 1:1 还原，`findHexAt` 命中计算保持单坐标系、改动最小。
- **全屏自适应（拒绝）**：画布随设备宽高动态计算 hex size。最贴近原生观感，但坐标数学和点击命中要全部响应式化，复杂度高。
- **固定宽度可变高度（拒绝）**：介于两者之间，地图垂直方向需滚动或留白。

## 核心决议

- **交付形态**：竖屏移动端，固定逻辑分辨率 **540×960**，letterbox 等比缩放。
- **HUD 渲染**：顶部双方栏（头像 + 地块数 + 人口）、中央倒计时、底部金币，**全部用 `ctx` 画进 canvas**，与地图、升级面板共享同一坐标系，不用 DOM 浮层。
- **输入**：只支持单击（tap），无拖拽/缩放；页面层禁用移动端默认手势（`touch-action:none`、viewport `user-scalable=no`、双击不缩放）。
- **升级面板**：从 DOM 浮层改为**纯 canvas 绘制**，需自行实现按钮命中区域与文字排版。
- **新增机制——人口上限**：见 `CONTEXT.md → 人口`。每方场上移动单位数量有硬上限（开发期 30，参考图为 100，建筑不计入）；满员时所有产兵源 CD 暂停，名额释放后恢复；双方独立，AI 同样受限。
- **图标体系不变**：参考图仅作画风/排版参考，可开格子仍用现有四图标语义（?/头盔/盾牌/金属），混合池与二次概率表不变。
- **砍掉**：聊天按钮、小地图——PvE 单人 + 单屏全可见，二者零玩法价值。

## Consequences

- **`AGENTS.md → Out of Scope` 的"mobile adaptation"作废**，需同步更新该文件与 Key Decisions。
- **人口上限需要重新平衡**：吐兵节奏、战斗规模、AI"满员不再开兵营"的决策都受影响。先用 30 跑起来再调。
- **升级面板重写**：从 DOM `#upgrade-modal` 迁到 canvas，要新增独立的点击区域命中系统（不能复用 `findHexAt`）。
- **HUD 从文字升级为图形**：头像需在单文件零依赖约束下内联（data URI / emoji / 程序化绘制），不能外链图片。
- **现有横版 560×540 渲染与点击坐标全部重定**到 540×960 单坐标系。
